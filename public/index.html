A-Frame takes the pain out of 3D on the web.

Virtual and mixed reality experiences are becoming mainstream.

Huge array of devices can run these.

Usually, built in 3D game engine like Unity (written in C#) or Unreal (written in C++).

Enter the web

Low-to-no cost of entry

With WebGL, WebGL 2 and WebAssembly, the web is stealthily becoming the premier place to launch 3D interactive experiences.

Lots of libraries out there to build these kinds of experiences in:

  • Three.js
  • Babylon.js
  • Phaser
  • PlayCanvas
  • A-Frame

A-Frame facts

Created by Mozilla at the end of 2015

Open-source, version 0.5.0 as of June 2017.

Focused on creating virtual reality experiences

An example A-Frame scene 


  <a-scene>
    <a-assets>
      <video id="video-texture" src="assets/test-video.mp4" autoplay muted 
      crossorigin playsinline -webkit-playsinline></video>
    </a-assets>
    <a-videosphere src="#video-texture" rotation="0 255 0"></a-videosphere>
  </a-scene>

How it works

HTML5 Custom elements (through a polyfill)

Allow you to write HTML-like syntax for 3D objects on the web.

Can use stadard DOM selection APIs like document.querySelector (or jQuery!) to manipulate 3D objects.

A layer of abstraction over three.js

Instead of writing an enormous amount of three.js boilerplate, A-Frame's HTML-like syntax makes it a breeze to build.

Use attributes to set values like rotation, position and scale easily.

Everything kept in memory

A-Frame keeps its changes in memory to avoid the performance hit of writing to the DOM.
